# 🎙️ Podcast Script: 웹 통신의 큰 흐름 (01_web_flow)

## 🎯 Introduction

오늘의 복습 주제는 **'웹 통신의 큰 흐름'**입니다. 주소창에 `google.com`을 입력했을 때, 화면에 페이지가 뜨기까지 어떤 마법 같은 일들이 벌어지는지 시니어 엔지니어의 시각에서 핵심만 짚어보겠습니다.

---

## 🏗️ Section 1: 주소 입력에서 IP 획득까지 (The Quest for IP)

1. **브라우저의 첫 번째 행동**:
   - 사용자가 URL을 입력하면 브라우저는 가장 먼저 **캐시**를 뒤집니다. 브라우저 캐시, OS 캐시, 라우터 캐시를 순서대로 확인해서 이전에 방문한 적이 있는지 봅니다.
2. **DNS(Domain Name System) 탐색**:
   - 캐시에 없다면 DNS 서버에 물어봅니다. 여기서 중요한 건 **'계층적 탐색'**입니다. Root -> TLD(.com) -> Authoritative DNS 순서대로 타고 내려가며 최종적인 IP 주소를 받아옵니다.

## 📦 Section 2: 데이터의 포장과 전달 (Encapsulation)

1. **데이터 작성 (L7 Application)**:
   - '구글 페이지를 줘!'라는 **HTTP Request Message**가 생성됩니다. (편지 작성)
2. **전송 준비 (L4 Transport)**:
   - TCP가 이 메시지를 받아서 **Segment**로 나눕니다. 여기서 포트 번호(Port 80/443)라는 라벨이 붙습니다. (소포 포장 시작)
3. **주소 기입 (L3 Network)**:
   - IP 계층에서 출발지/목적지 IP가 적힌 **Packet**이 됩니다. (송장 부착)
4. **운송장 및 물리 전달 (L2/L1)**:
   - MAC 주소가 기록된 **Frame**이 되어 실제 전기 신호나 광신호로 변환되어 전선(L1)을 타고 서버로 날아갑니다.

## 🎖️ Section 3: 시니어의 통찰 (Senior Insight)

- **OSI 7계층 군대 소포 비유**:
  - L7은 편지 내용, L4는 소포 상자, L3는 택배 운송장, L2는 배송 트럭 번호판으로 이해하면 쉽습니다.
- **핵심 키워드**:
  - **계층적 탐색**: DNS의 구조적 조회 방식.
  - **데이터 캡슐화(Encapsulation)**: 상위 계층의 데이터가 하위 계층으로 내려가며 헤더가 붙는 과정. 역캡슐화는 그 반대.

---

## 🏆 Summary

웹 통신은 단순히 선이 연결되는 것이 아니라, **'계층적 탐색'**과 **'데이터의 캡슐화'**라는 두 개의 톱니바퀴가 완벽하게 맞물려 돌아가는 장엄한 기계 장치와 같습니다.
