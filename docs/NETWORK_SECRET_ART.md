# 📔 시니어 면접관의 네트워크 비급 (Senior's Network Insight)

_본 문서는 NotebookLM을 통해 집약된 시니어 면접관의 관점과 통찰을 정제한 비급입니다._

---

## 🏛️ 1. 계층 모델의 전략적 가치 (L1-L7)

- **핵심:** 계층화는 단순히 분류가 아니라 **'문제 해결의 추상화'**다.
- **실전 통찰:** "Ping은 가는데 웹이 안 뜬다?" → L3(IP) OK, L7(HTTP) Issue.
- **IaC 연결:** AWS Security Group 설정은 L4, VPC Peering은 L3의 도메인이다.

## 🔗 2. 하드웨어와 주소 체계 (L1 & L2)

- **NIC의 방어:** 목적지 MAC 주소가 자신이 아니면 **Discard(폐기)**하여 CPU 자원을 보호한다.
- **허브 vs 스위치:** 스위치는 '주소 인식 능력'을 통해 **Collision Domain(충돌 영역)**을 포트별로 분리한다.
- **CORS의 진실:** 네트워크 연결은 하위 계층에서 성공해도, CORS라는 보안 정책은 **L7(응용 계층)**에서 브라우저에 의해 강제된다.

## 🗺️ 3. 최적 경로 탐색 (L3)

- **서브네팅:** 비트 AND 연산을 통해 Network ID를 도출한다. (예: `/25`는 상위 25비트가 네트워크 영역)
- **ARP 3단계:** Request(브로드캐스트) → Response(유니캐스트) → Update(테이블 갱신).
- **Default Route (0.0.0.0/0):** 최후의 수단 게이트웨이. 클라우드 IGW 설정의 핵심.

## 🛡️ 4. 신뢰성과 흐름 제어 (L4)

- **TCP:** 비즈니스 결제 시스템 등 1비트의 오류도 허용되지 않을 때 필연적 선택.
- **UDP:** 1ms의 지연시간이 매출로 직결되는 실시간 트레이딩, 스트리밍에 적합.
- **Handshake:** 3-way(연결 수립), 4-way(연결 해제)의 절차적 완결성.

## 🌐 5. HTTP 심화 (L5-L7)

- **HTTPS:** 비대칭키(키 교환) + 대칭키(데이터 암호화)의 복합 알고리즘.
- **Stateless & Idempotency:** REST API의 핵심. 수평 확장(Scaling)을 가능케 하는 근간.

## 🔱 꼬리 질문 역공용 보급 (Advanced Counter-Attack)

1.  **캡슐화의 디테일:** Data(L7) → Segment(L4) → Packet(L3) → Frame(L2) → Bits(L1). 각 명칭(PDU)을 정확히 사용하는 것이 엔지니어의 증거.
2.  **병목 분석:**
    - DNS 지연 (첫 패킷 전 지연)
    - TCP 패킷 손실 (재전송에 따른 Throughput 급감)
    - L7 오버헤드 (HTTPS 연산 및 무거운 헤더 부하)
